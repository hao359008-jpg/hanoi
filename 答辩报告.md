# 汉诺塔游戏项目报告

## 一、项目概述

### 1.1 项目背景
汉诺塔（Tower of Hanoi）是一个经典的数学益智游戏，源于印度传说。本项目基于C++语言实现了一个完整的汉诺塔游戏系统，包含手动游戏模式、自动演示模式、游戏存档等功能。

### 1.2 项目目标
- 实现完整的汉诺塔游戏逻辑
- 提供友好的控制台用户界面
- 支持递归和非递归两种算法演示
- 实现游戏进度的保存与加载功能
- 支持难度设置（1-10层）

### 1.3 技术栈
- **语言**: C++11/14
- **开发环境**: Visual Studio Code
- **构建工具**: MSVC / MinGW

```
recursiveHelper(n, from, to, aux)
│
├─ if n == 0: return
│
├─ recursiveHelper(n-1, from, aux, to)  // 把n-1个从from移到aux
│
├─ 记录移动 (from → to)                   // 把第n个从from移到to
│
└─ recursiveHelper(n-1, aux, to, from)  // 把n-1个从aux移到to
```

## 四、核心实现

### 4.1 递归算法实现

```cpp
void HanoiGame::recursiveHelper(int n, int from, int to, int aux, 
                                vector<pair<int, int>>& moves) const {
    if (n == 0) return;
    recursiveHelper(n - 1, from, aux, to, moves);   // 步骤1: 移n-1个到辅助柱
    moves.emplace_back(from, to);                   // 步骤2: 移最大盘到目标柱
    recursiveHelper(n - 1, aux, to, from, moves);   // 步骤3: 移n-1个到目标柱
}
```

**算法复杂度**: 时间复杂度 O(2ⁿ)，空间复杂度 O(n)

### 4.2 非递归算法实现

非递归算法基于以下规律：
- 当盘子数为奇数时，最小盘每次顺时针移动
- 当盘子数为偶数时，最小盘每次逆时针移动
- 其他步骤在非最小盘所在的两个柱子间进行合法移动

```cpp
vector<pair<int, int>> HanoiGame::generateNonRecursiveMoves() const {
    vector<pair<int, int>> moves;
    vector<int> pillars[3];
    for (int i = m_diskCount; i >= 1; --i) pillars[0].push_back(i);
    
    int minPos = 0;
    int totalSteps = getOptimalSteps();

    for (int i = 0; i < totalSteps; ++i) {
        if (i % 2 == 0) {
            // 移动最小盘
            int nextMinPos = (m_diskCount % 2 == 1) ? (minPos + 2) % 3 : (minPos + 1) % 3;
            pillars[minPos].pop_back();
            pillars[nextMinPos].push_back(1);
            moves.emplace_back(minPos + 1, nextMinPos + 1);
            minPos = nextMinPos;
        } else {
            // 在另外两个柱子间移动
            // ... 选择合法移动
        }
    }
    return moves;
}
```

**算法复杂度**: 时间复杂度 O(2ⁿ)，空间复杂度 O(n)

### 4.3 移动合法性验证

```cpp
bool HanoiGame::checkMoveValid(int fromPillar, int toPillar) const {
    if (fromPillar < 1 || fromPillar > 3 || toPillar < 1 || toPillar > 3) 
        return false;  // 柱子编号无效
    if (fromPillar == toPillar) 
        return false;  // 相同柱子
    int fromIdx = fromPillar - 1, toIdx = toPillar - 1;
    if (m_pillars[fromIdx].isEmpty()) 
        return false;  // 起始柱为空
    int fromTop = m_pillars[fromIdx].getTopSize();
    int toTop = m_pillars[toIdx].getTopSize();
    return toTop == 0 || fromTop < toTop;  // 目标柱为空或顶部盘子更大
}
```

## 六、技术亮点

### 6.1 设计模式应用

1. **策略模式**: 通过 `GameBase` 抽象基类定义接口，`HanoiGame` 实现具体逻辑，便于扩展其他游戏类型。

2. **组合模式**: `HanoiGame` 包含 `Pillar` 对象，`Pillar` 包含 `Disk` 对象，形成树形结构。

### 6.2 算法实现对比

| 特性 | 递归算法 | 非递归算法 |

| 代码复杂度 | 简单直观 | 相对复杂 |
| 空间复杂度 | O(n) 栈空间 | O(n) 堆空间 |
| 适用场景 | 教学演示 | 大规模计算 |
